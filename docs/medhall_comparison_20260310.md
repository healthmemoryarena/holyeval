## 六、Theta Agents vs LLM 基模 — 对比总结

| 维度 | Theta Agents 优势 | LLM 基模优势 |
|------|-------------------|-------------|
| **事实幻觉** | 全部 100% 通过，医学知识可靠 | — |
| **上下文幻觉** | — | GPT-5.2 达 100%，能更准确回忆用户档案细节 |
| **引用幻觉** | Miroflow 90% 领先所有系统 | — |
| **日期/时间感知** | 3/3 系统通过日期感知 case | GPT-4.1 完全失败（报告错误年份） |
| **健康数据忠实度** | — | GPT-4.1/5.2 在时间和部位混淆上更准确 |
| **执行速度** | — | GPT-5.2 最快（1048s），Expert 耗时近 2 倍 |

### 核心发现

1. **引用幻觉是全行业难题**：通过率最高的 Miroflow 也仅 90%，GPT-4.1/5.2 仅 50%。当要求引用具体文献时，所有 LLM 都有显著的捏造倾向。

2. **Theta agents 在事实类表现突出**：三个 agent 在 12 个事实幻觉 case 上全部通过，而 GPT-4.1 和 GPT-5.2 分别出现 2 个和 1 个失败。

3. **上下文幻觉呈现反转格局**：Theta agents 因依赖真实用户档案 API，在「时间混淆」「部位混淆」类 case 上反而不如直接使用对话上下文的 LLM 基模。这可能是 Theta 数据检索层面的问题，而非推理能力差异。

4. **Miroflow 综合最优**：通过率最高（89.2%），引用幻觉表现远超其他系统，但在上下文忠实度上有改进空间。

5. **GPT-4.1 综合最弱**：8 个失败 case 覆盖所有幻觉类型，日期感知和健康数据回忆尤其薄弱。

## 八、建议

1. **引用幻觉缓解**：考虑引入 RAG 管线或引用验证后处理（DOI/PMID 校验），减少捏造文献
2. **上下文幻觉修复（Theta）**：排查 `mt_con_0021`（时间混淆）和 `mt_con_0022`（部位混淆）的数据检索逻辑，确认是否为 API 返回数据不完整导致
3. **扩大评测规模**：当前 37 case 样本较小，建议扩充至 100+ case 以获得更稳健的统计结论
4. **引入 Miroflow 优势**：Miroflow 在引用幻觉上的优势值得分析其 prompt/检索策略，可能可迁移到其他 agent
